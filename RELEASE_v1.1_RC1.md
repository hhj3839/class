# 우리반 이음 v1.1.0-rc.1

릴리스 후보 기준일: 2026-07-20  
배포 주소: https://hhj3839.github.io/class/

## 릴리스 목적

PRD v1.1의 개발 범위를 고정하고, 8월 실제 교실 태블릿 파일럿에서 검증할 기준 버전을 남긴다. AI 결과는 학생에 대한 판정이 아니라 담임교사가 사실을 확인하기 위한 보조 자료로만 사용한다.

## 포함 기능

- 교사 회원가입, 이메일 인증, 로그인, 비밀번호 재설정
- 브라우저 창 단위 교사 세션과 만료 전 자동 갱신
- 담당 학급별 권한, 학생 명단, 참여 토큰과 QR
- 월별 설문 제출 현황과 응답 정정·분석 제외
- 월별·누적 분석, 학생 상세, 관계망 신뢰도와 변화
- 서버 측 AI 분석, 근거 원문 연결, 교사 유용성 평가
- 관찰 확인 결과와 파일럿 운영 지표
- 보존 정책, 감사 로그, 소프트 삭제, 백업·복구
- AI 분석 PDF 다운로드
- 배포·DB·기기 호환성 파일럿 사전점검
- 학생 개인정보가 없는 파일럿 점검 결과 다운로드

## 검증 완료

- Node 자동 테스트 55개 통과
- GitHub Pages 최신 자산 버전 로드 확인
- 운영 교사 계정에서 담당 학급 권한과 데이터 조회 확인
- 파일럿 준비 상태 13/13 항목 통과
- 학생 이름·응답 원문·교사 이메일·학급 ID를 점검 보고서에서 제외
- 교사용 민감정보는 로그인 전 렌더링하지 않고 `localStorage`에 저장하지 않음

## 파일럿에서 확인할 항목

다음 항목은 개발 완료가 아니라 실제 교실 환경에서 확인해야 한다.

1. 서로 다른 태블릿 3~5대에서 QR 접속과 이름 선택
2. 설문 작성 중 화면 회전, 잠금 후 복귀, 네트워크 재연결
3. 동시 제출 시 누락·중복 여부와 제출 완료 안내
4. 평균 작성 시간과 학생이 어려워하는 문항
5. 교사 화면의 제출 현황 반영 시간
6. AI 추천 3명의 근거와 실제 담임 관찰의 일치 여부
7. PDF 파일 다운로드와 학교 기기에서의 한글 표시
8. 파일럿 종료 후 백업 생성과 복구 가능 여부

## 통과 기준

- 설문 완료율 90% 이상
- 치명적 오류 0건
- 다른 학급·다른 교사의 자료 노출 0건
- AI 추천 검토 기록 누락 0건
- 학생 이름과 응답 원문이 점검 파일에 포함되는 사례 0건

## 배포 및 복구 기준

- 배포 기준 브랜치: `main`
- 릴리스 후보 태그: `v1.1.0-rc.1`
- 기능 변경은 별도 브랜치에서 검증한 뒤 `main`에 반영한다.
- 파일럿 중 치명적 회귀가 발생하면 이 태그의 코드와 직전 Supabase 백업을 복구 기준으로 사용한다.

## 주의

Git 태그는 프런트엔드 코드의 기준점을 보존한다. Supabase 데이터와 설정을 자동으로 되돌리지는 않으므로, SQL 변경 전에는 별도의 학급 백업과 적용 기록을 남겨야 한다.
