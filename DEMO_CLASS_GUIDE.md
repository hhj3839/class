# 기능 점검용 데모 학급 사용 안내

실제 학생 자료 없이 시험용 교사 계정으로 주요 구현 화면을 확인하기 위한 비식별 데이터 세트다. 학생 이름과 응답은 모두 가상이며 실제 학생 판단이나 연수 자료로 사용하지 않는다.

## 생성

1. Supabase Dashboard의 **SQL Editor → New query**를 연다.
2. `supabase/demo/seed_demo_class.sql` 전체를 붙여 넣는다.
3. 첫 부분의 `REPLACE_WITH_TEST_TEACHER_EMAIL`을 기존 시험용 교사 이메일로 바꾼다.
4. **Run**을 누른다.
5. 시험용 교사 계정으로 웹앱에 로그인하고 `기능 점검 교사 · 5학년 99반` 학급을 선택한다.

스크립트를 다시 실행하면 해당 계정의 고정 `demo-*` 학급만 초기화한다. 같은 교사의 다른 운영 학급은 변경하지 않는다.

## 포함된 점검 상황

- 가상 학생 10명과 최근 3개월 응답
- 최신 달 미제출 학생 1명과 추가 제출 1건
- 분석 제외 응답 1건
- 직접 도움 요청, 이번 주 대화 요청, 종결된 확인 기록
- 설문이 없는 학생의 진행 중 확인·지원 기록
- 관계 점수 상승·하락과 구조화된 긍정 언급
- 세 개의 느슨한 관계 묶음, 묶음 사이 연결 학생, 비대칭 관계와 친구별 응답 편차
- 최근 3개월 동안 강화되는 관계와 약화되는 관계
- API 호출 없이 확인 가능한 저장 AI 분석 결과 1건
- 진행 중 및 완료된 관찰 기록

## 화면별 빠른 확인

| 화면 | 확인할 내용 |
|---|---|
| 설문 관리 | 최신 달 제출 9/10명, 미제출 윤새봄, 오가람 추가 제출 |
| 학급 분석 | 저장된 AI 결과, 먼저 살펴볼 학생 3명, 학급 패턴 |
| 학생별 살펴보기 | 김하늘 상승, 이하람 변화, 윤새봄 무응답+지원 기록 |
| 관계 분석 | 세 관계 묶음, 묶음 사이 연결, 제한된 연결, 비대칭과 월별 변화가 섞인 3개월 누적 관계망 |
| PDF | 학급 AI 분석과 학생 기록 다운로드 |
| 권한 | 다른 시험 교사 계정에서 이 학급이 보이지 않는지 확인 |

AI의 `학생-4` 같은 익명 번호는 교사 화면에서 가상 학생 이름으로 치환된다. 새 분석을 누르면 실제 API 사용량에 포함되므로 저장 결과 확인만 필요할 때는 누르지 않는다.

## 삭제

`supabase/demo/reset_demo_class.sql`의 이메일을 같은 시험용 교사 이메일로 바꾸고 SQL Editor에서 실행한다. 해당 시험용 계정의 고정 데모 학급과 종속 데이터만 삭제된다.

운영 학급을 삭제 대상으로 직접 입력하는 방식은 제공하지 않는다.
