# 우리반 이음 PRD v1.2.1

- 문서 버전: 1.2.1
- 작성일: 2026-07-22
- 제품 단계: v1.2.1-prepilot 기준선 확정, 현장 파일럿 대기
- 배포 주소: https://hhj3839.github.io/class/
- 이전 기준: `RELEASE_v1.1_RC2.md`

## 1. 제품 정의

우리반 이음은 학생의 월별 학교생활·교우관계 설문을 담임교사의 실제 관찰과 대화로 연결하는 교사용 웹앱이다. v1.2의 핵심은 분석 결과를 보여주는 데서 끝내지 않고, `응답 → 확인 → 지원 → 결과 기록`이 끊기지 않는 담임 업무 흐름을 만드는 것이다.

이 제품은 학생을 진단하거나 위험 학생으로 분류하지 않는다. 규칙 기반 신호, 관계망, AI 분석과 우선순위는 모두 교사가 먼저 확인할 항목을 정리하는 보조 수단이며, 사실 판단은 설문 원문·학생 대화·교사의 직접 관찰을 통해서만 이루어진다.

## 2. 해결하려는 문제

1. 월별 설문을 받아도 어떤 학생을 먼저 확인해야 하는지 한눈에 파악하기 어렵다.
2. 설문 신호, AI 분석, 관찰 기록이 서로 분리되면 후속 확인이 누락될 수 있다.
3. 관계 점수와 서술 응답을 단정적으로 해석하면 학생을 낙인찍을 위험이 있다.
4. 학생 개인정보가 공개 저장소, 로그인 전 화면 또는 다른 교사 계정에 노출될 수 있다.
5. 담임의 지원이 실제로 확인·종결되었는지 운영 결과를 남기기 어렵다.

## 3. 목표와 비목표

### 3.1 목표

- 학생이 약 10분 안에 태블릿으로 월별 설문을 제출한다.
- 담임은 제출 현황과 미제출 학생을 월별로 확인한다.
- 담임은 확인이 필요한 학생과 근거를 학생 단위로 파악한다.
- 안전 신호를 상태·후속 확인일·교사 메모와 함께 관리한다.
- 학생 상세에서 설문, 안전 신호, 관찰과 지원 결과를 시간순으로 확인한다.
- 모든 민감 데이터 조회·변경에 담당 학급 권한을 적용한다.
- AI 결과에는 근거와 한계를 함께 표시하고 실제 관찰을 대신하지 않도록 한다.

### 3.2 비목표

- 폭력, 고립, 정신건강 상태의 자동 진단
- 학생 서열화 또는 생활평가 점수 산출
- 관계망을 실제 친한 무리나 고정 집단으로 확정
- AI의 자동 상담·징계·보호자 연락 결정
- 다른 교사 또는 학교 간 학생 데이터 공유
- 생체정보, 위치정보, 연락처 수집

## 4. 주요 사용자

### 담임교사

- 학급과 학생 명단을 관리한다.
- 설문 링크와 QR을 배포한다.
- 응답·관계 변화·AI 분석을 검토한다.
- 학생과 대화하고 실제 장면을 관찰한다.
- 확인 상태, 관찰 사실과 지원 결과를 기록한다.

### 학생

- 교사가 공유한 임의 참여 링크로 접속한다.
- 이름을 선택하고 월별 설문에 응답한다.
- 자신의 고민과 도움 요청을 직접 작성한다.
- 다른 학생의 기존 응답이나 교사용 분석에는 접근할 수 없다.

## 5. 핵심 사용자 흐름

1. 교사가 로그인하고 담당 학급을 불러온다.
2. 교사가 학생 설문 링크 또는 QR 코드를 공유한다.
3. 학생이 이름을 선택하고 설문을 제출한다.
4. 교사가 월별 제출 현황과 원문을 확인한다.
5. 시스템이 규칙 기반 신호, 누적 관계 자료와 AI 분석을 제시한다.
6. 교사가 학생별 살펴보기와 학급 분석에서 먼저 살펴볼 학생과 확인할 장면을 선택한다.
7. 교사가 대화·사실 확인·관찰을 진행하고 필요할 때 내부 기록으로 상태를 남긴다.
8. 필요한 경우 지원을 연결하거나 특이사항 없음으로 종결한다.
9. 학생 상세 타임라인에서 설문부터 확인 결과까지 연속성을 검토한다.

## 6. 기능 요구사항

상태 표기는 `완료`, `부분 완료`, `계획`으로 구분한다.

### F01. 인증과 학급 권한 — 완료

- 로그인 전에는 인증 화면만 표시한다.
- 교사용 메뉴와 학생 개인정보는 인증 후에만 렌더링한다.
- 교사는 자신에게 연결된 학급만 조회·수정할 수 있다.
- 로그아웃 또는 세션 만료 시 응답, 명단, 관찰과 분석 메모리를 즉시 삭제한다.
- 회원가입, 이메일 인증 복귀, 비밀번호 재설정을 지원한다.

### F02. 학생 명단과 참여 링크 — 완료

- 학생 이름과 번호를 Supabase에 저장한다.
- 공개 저장소에는 실제 학생 명단을 포함하지 않는다.
- 학생 링크는 학급 ID 대신 임의 참여 토큰을 사용한다.
- 교사는 QR 저장, 링크 복사와 링크 재발급을 할 수 있다.
- 링크 재발급 시 기존 링크는 폐기된다.

### F03. 월별 학생 설문 — 완료

- 확정된 1~9번 문항을 순서대로 표시한다.
- 2번과 5번 문항은 확정 원문을 변경하지 않는다.
- 8번에서 본인은 스마일 아이콘, 친구는 관계 점수를 선택한다.
- 작성 중 내용은 현재 탭의 세션 저장공간에만 임시 보관하고 제출 성공 시 삭제한다.
- 네트워크 재시도 시 동일 제출 ID가 중복 응답을 만들지 않는다.
- 월 1회 제한과 설문 기간 제한은 두지 않는다.

### F04. 설문 관리 — 완료

- 설문 관리 화면은 `학생 설문 링크`와 `제출 현황` 두 탭으로 구분한다.
- 응답 정정·분석 제외 기능과 감사 기록은 유지하지만 일상적인 담임 화면에서는 숨긴다.
- 월별 제출 인원, 미제출 학생과 추가 제출 건수를 확인한다.
- 응답 분석 제외와 소프트 삭제에는 사유와 감사 로그를 남긴다.

### F05. 규칙 기반 분석 — 완료

- 즉시 도움 요청, 교사 대화 요청, 자기평가 급변과 관계 응답의 복수 신호를 계산한다.
- 계산 결과, 학생 원문, 가능한 해석과 교사 확인 제안을 구분한다.
- 결석·미제출은 0점으로 계산하지 않고 평균 분모에서 제외한다.
- 분석 제외 응답은 계산에서 제외한다.

### F06. 안전 신호 확인함 — 완료

> 안전 신호 데이터·상태 관리 기능은 유지하지만 교사 홈과 기본 사이드바에서는 노출하지 않는다. 학생 상세 지원 타임라인과 내부 운영 흐름에서 사용한다.

- 상태는 `확인 전 → 대화 예정 → 사실 확인 중 → 지원 연결 / 특이사항 없음 / 종결`을 사용한다.
- 후속 확인일과 300자 이내 교사 메모를 저장한다.
- 확인 전, 진행 중과 기한 경과 건수를 표시한다.
- 확인 필요, 처리 완료와 전체 필터를 제공한다.
- 각 신호에서 연결된 근거를 확인한다.
- 상태 변경은 담당 교사 RPC를 사용하고 감사 로그를 남긴다.

### F07. 담임 확인 우선순위 — 완료

> 우선순위 계산은 내부 호환성을 위해 유지하지만 교사 홈 카드에서는 제거했다. 교사에게는 AI 분석의 확인 대상과 학생 상세의 객관적 요약을 우선 제공한다.

- 동일 학생의 미처리 안전 신호와 진행 중 관찰 기록을 한 카드로 묶는다.
- 즉시 확인 응답, 기한 경과, 미확인 상태를 먼저 정렬한다.
- 화면에는 위험도 순위가 아닌 업무상 확인 순서임을 표시한다.
- 카드에서 해당 학생 상세 화면으로 이동한다.
- 자동 징계·지원 결정에는 사용하지 않는다.

### F08. 학생별 살펴보기와 확인·지원 이력 — 완료

- 최초 진입 시 학생을 선택하기 전에는 개인정보를 표시하지 않는다.
- `한눈에 보기`, `변화 살펴보기`, `월별 응답`의 세 탭으로 구분한다.
- 학급 분석과 같은 도구 모음에서 `한눈에 보기`, `변화 살펴보기`, `월별 응답` 탭을 왼쪽에, 이전·다음 학생 이동·학생 선택·학생 기록 PDF 저장을 오른쪽에 제공한다.
- 한눈에 보기에는 최근 설문 제출, 친구들이 준 관계 점수 평균과 친절·성장 문항에서 이름이 언급된 객관적 건수만 표시한다.
- 지난 제출과 비교해 관계 점수 평균이 0.5점 이상 달라졌거나 친절·성장 문항 이름 언급 건수가 달라졌을 때만 `지난 제출과 달라진 점`을 표시한다.
- 자기평가 변화는 변화 살펴보기에서, 학생 원문은 월별 응답에서 확인한다.
- 월별 응답 기록은 모든 월을 기본으로 접고, 교사가 선택한 월만 펼쳐서 보도록 한다.
- 설문 제출, 확인이 필요한 응답 상태와 교사 확인 결과를 하나의 확인·지원 이력으로 연결한다.
- 확인·지원 이력은 변화 살펴보기 탭에서 확인 진행 중 항목 수와 다음 확인 예정일만 먼저 표시하고, 상세 이력은 기본으로 접는다. 연결된 확인·지원 기록이 없으면 해당 영역을 숨긴다.
- 확인·지원 이력은 월과 기록 종류(설문 제출·확인이 필요한 응답·교사 확인 기록)로 필터링한다.
- 학생별 살펴보기에서 학생 기록 PDF를 바로 저장하며 최근 확인·지원 이력 최대 8건과 종류별 건수를 포함한다.

### F09. 누적 관계 분석 — 완료

- 전체 월의 실제 응답만 누적 평균한다.
- 선택한 달의 설문 참여율과 친구 관계 응답률이 각각 80% 이상일 때 `월별 관계 지도`를 표시한다.
- 화면은 `표시 방법`, `월별 관계 지도`, `AI 관계 코칭`의 세 탭으로 구분한다.
- 학급 분석과 같은 상단 도구 모음의 오른쪽에서 월을 선택한다. 같은 드롭다운의 첫 항목인 `여러 달 함께 보기`를 고르면 전체 월의 실제 응답 평균을 비교하고, 특정 월을 고르면 해당 월의 관계 지도로 돌아간다. 월별 AI 관계 코칭 탭에서는 월 항목만 제공한다.
- 지난달과 비교해 새로 확인된 상호 연결, 이어서 확인된 상호 연결과 이번 달 응답에서 확인되지 않은 연결의 수를 표시한다. 연결이 확인되지 않은 것을 관계 단절로 표현하지 않는다.
- 관계 지도에서는 학생 이름이 서로 겹치지 않도록 배치하고, 관계 선·점선·색상의 의미를 별도 범례로 제공한다.
- AI 관계 코칭에서는 교사가 선택한 달의 관계 응답을 중심으로, 이전 달 자료가 있을 때만 변화를 비교해 실제 학생 이름을 포함한 핵심 관계 모습을 제공한다.
- AI 관계 코칭은 전체 요약 한 문장과 겹치지 않는 핵심 관계 모습 최대 2개만 제공한다. 각 관계 모습은 `이번 달에 보이는 변화`, `여러 달 이어진 관계 모습`, `응답이 적어 추가 확인 필요` 중 하나의 근거 범위를 표시하고, 50자 이내 한 문장의 `교실에서 살펴볼 점`과 `학급 코칭`만 덧붙인다. 상세 수치는 접힌 `분석 근거`에서 확인한다.
- 이번 달에 연결이 확인되지 않았다는 이유만으로 관계 단절로 해석하지 않는다.
- 관계 AI에는 이름 대신 익명 번호와 선택한 달의 관계 계산 결과, 비교 가능한 이전 달 요약만 전달하고 교사 화면에서 `학생-1`, `학생-1번`, `1번 학생` 등 표현을 실제 이름으로 치환한다.
- 전학 등으로 현재 명단에서 제외된 학생의 과거 응답은 보존하되, 새 관계 AI 분석 입력에서는 현재 활성 명단의 응답자와 평가 대상만 사용한다.
- 이전 저장 결과에 현재 명단에 없는 익명 번호가 있으면 내부 번호를 노출하지 않고 `이전 명단 학생`으로 표시하며, 현재 명단 기준 AI 새 분석을 안내한다.
- 이전 형식으로 저장된 관계 AI 결과는 그대로 열람할 수 있게 유지하되, 선택한 달 기준의 새 분석을 권장한다.
- 추정 집단, 집단 연결 역할, 누적 데이터 충족도와 전월 연결 변화 같은 복잡한 지표는 기본 화면에 노출하지 않는다.
- 관계망은 고정 집단 또는 고립 판정이 아님을 표시한다.

### F10. 관찰 기록과 결과 환류 — 완료

> 관찰 기록 저장·연결·감사 기능은 유지하지만 독립 사이드바 메뉴는 제공하지 않는다. 학생 상세 지원 타임라인을 통해 맥락을 확인한다.

- 관찰 예정, 관찰 중, 확인 완료 상태를 제공한다.
- 관찰 사실, 학생과 나눈 이야기, 교사 해석, 후속 조치와 확인일을 구분한다.
- 확인 완료 시 관찰 사실과 확인 결과를 필수로 요구한다.
- 결과는 계속 관찰, 지원 필요와 특이사항 없음으로 구분한다.
- 확인 완료 시 종결 사유를 필수로 기록하고, 후속 확인 결과와 추가 확인 필요 여부를 구조화해 저장한다. 추가 확인이 필요하면 후속 확인일을 함께 지정한다.
- 삭제는 소프트 삭제로 처리하고 감사 로그를 남긴다.

### F11. 서버 측 AI 분석 — 완료

- OpenAI API 키는 Supabase Edge Function 비밀값에만 저장한다.
- 브라우저와 GitHub 저장소에는 API 키를 저장하지 않는다.
- AI에는 학생 이름 대신 익명 번호를 전송하고 교사 화면에서만 이름으로 치환한다.
- 익명 번호와 실제 이름 또는 `개별 학생`을 치환할 때는 이름의 받침에 맞춰 한국어 조사를 함께 보정한다.
- 우리 반 응답 흐름이 특정 학생의 응답을 근거로 들 때는 AI가 `학생-N` 식별자를 유지해야 하며, 누구인지 복원할 수 없는 `개별 학생`·`한 학생` 표현으로 바꾸지 않는다. 교사 화면에서만 해당 식별자를 현재 명단의 실명으로 치환한다.
- 학급 분석은 `우리 반 응답 흐름`과 `AI 학생 지원`으로 역할을 구분한다.
- `우리 반 응답 흐름`은 설문 참여, 학교생활 자기평가, 친구 관계 응답, 교사와 대화 희망, 바로 확인할 응답을 규칙 기반 수치로 항상 표시한다. 같은 AI 호출 결과에서 학생 이름을 제거한 `이번 달 우리 반에서 보이는 흐름`을 보조 정보로 제공하며 별도 호출 한도를 사용하지 않는다.
- 받은 평가, 서술 응답과 월별 변화를 근거로 `AI 학생 지원`에서 최대 3명의 확인 대상을 제시한다.
- 기본 결과 화면은 학생별 `담임 참고 → 교실에서 살펴볼 점 → 학생 코칭`의 세 항목만 표시한다. `교실에서 살펴볼 점`의 장면·행동·시기는 `/`로 구분해 빠르게 읽을 수 있게 하고, 학생 코칭은 담임이 학생에게 비공개로 실제 건넬 수 있는 말로 작성한다. `분석 내용 문제 표시` 버튼은 기본 화면에서 제거하되 기존 감사 구조는 보존한다.
- 학급 분석에는 저장 결과와 새 분석 모두 마지막 분석 날짜·시간을 한국 시간으로 표시한다.
- AI 학생 지원과 AI 관계 코칭은 저장 유형과 저장 결과를 서로 분리하며, 월별 새 분석 횟수 제한은 두 기능 모두 적용하지 않는다.
- AI 관계 코칭에도 AI 학생 지원과 같은 `저장된 결과 불러오기`, `AI 새 분석`과 마지막 분석 날짜·시간 표시를 제공한다.
- 긍정 요소는 자료에서 확인될 때 `참고사항` 한 문장으로 표시한다. 다음 지원 방향·피하면 좋은 대응·일반적인 분석 한계는 기본 카드에서 제거하고 필요한 근거만 접힌 영역에서 확인한다.
- 학생별 관찰 포인트, 코칭 방향과 질문은 각각 최대 2개로 제한한다.
- 학생 원문과 계산 근거는 기본 화면에서 분리하되 필요한 경우 검증 가능한 원문으로 이동할 수 있어야 한다.
- AI 근거에서 허용된 응답 ID와 문항 원문으로 이동한다.
- 새 분석 횟수는 제한하지 않되 저장된 결과 재사용, 실행 기록, 실패 복구와 감사 로그를 유지한다. 호출할 때마다 외부 AI 사용 비용이 발생할 수 있음을 운영자가 인지해야 한다.
- AI 결과는 진단이나 사실 판정이 아님을 항상 표시한다.
- `우리 반 응답 흐름`은 AI 학생 지원과 같은 저장 결과 불러오기·AI 새 분석·마지막 분석 시점을 제공한다. AI가 학급 전체의 인원·평균·변화를 중심으로 최대 3개 흐름을 만들고, 각 항목의 `분석 근거`와 60자 이내 한 문장의 `교실에서 살펴볼 점`을 보여준다. 특정 학생은 Edge Function에 익명 번호로 전달하고 교사 전용 화면에서 실제 이름으로 치환한다.

### F12. PDF 보고서 — 부분 완료

- 학급 분석의 `분석 결과 PDF 저장`은 별도 옵션 창 없이 선택한 달의 AI 학생 지원 결과를 PDF로 다운로드한다.
- 학생별 살펴보기의 `학생 기록 PDF 저장`은 AI 분석이 없어도 학생 요약, 월별 변화·응답과 최근 확인·지원 이력을 다운로드한다. 선택한 달의 해당 학생 AI 결과가 있으면 참고 내용으로 추가한다.
- 브라우저 폰트 로딩 완료 후 캔버스로 렌더링하며 파일명에 사용할 수 없는 문자를 제거한다.
- A4 여백 안에서 제목·표·학생별 분석 블록을 단위별로 배치하고, 긴 블록만 페이지로 나누며 페이지 번호를 표시한다.
- 학교 기기에서 한글, 페이지 분할과 다운로드 성공 여부는 현장 검증이 필요하다.
- 학생 개인 PDF에는 AI 분석, 월별 추세와 최근 지원 타임라인 요약을 함께 담는다.
- 긴 학생 이름·긴 한글 문장·다중 페이지·지원 이력 최대 8건을 기준으로 자동 PDF 사전 점검을 수행한다.

### F13. 데이터 거버넌스 — 완료

> 보존 정책·감사 로그·파일럿 준비 기능은 유지하지만 일반 학급 설정 화면에서는 숨긴다.

- 응답 정정·분석 제외·삭제·관찰 저장·안전 신호 상태 변경·보존 정책 변경을 감사 로그에 남긴다.
- 보존 기간과 학년 말 처리 계획을 설정한다.
- JSON 백업과 권한 검사된 복구를 지원한다.
- 다른 학급으로 복구할 때 학생과 응답 UUID를 새로 매핑한다.
- 자동 영구 삭제는 수행하지 않는다.

### F14. 운영 품질과 파일럿 측정 — 부분 완료

- 최근 참여율, 평균 설문 완료 시간과 관찰 결과를 집계한다. 과거 AI 유용성 평가 기록은 감사 호환성을 위해 보존하지만 현재 화면에서 새 평가를 받지 않는다.
- 개인정보를 제외한 파일럿 준비 보고서를 내려받는다.
- 화면 숨김과 페이지 이탈 직전 임시 저장, 연결 복구 시 명단 재조회와 제출 중복 방지를 자동 검증한다.
- 서로 다른 실제 태블릿의 동시 제출·회전·잠금·네트워크 재연결 시험은 2026년 8월 중순 이후 진행한다.
- 학급 지원 환류 지표에는 안전 신호 전체 건수 대비 확인률과 처리 완료율을 포함한다.
- 시험용 교사 계정에는 운영 학급과 분리된 비식별 데모 학급을 생성해 주요 화면·빈 상태·확인 흐름·저장 AI 결과를 점검할 수 있다.

## 7. 데이터와 권한 원칙

### 7.1 데이터 경계

- GitHub: 공개 가능한 소스 코드, 비식별 테스트와 SQL 마이그레이션만 저장
- Supabase: 학급, 학생, 응답, 관찰, 분석, 감사 로그와 보존 정책 저장
- Edge Function Secret: `OPENAI_API_KEY` 저장
- 브라우저: 교사용 개인정보를 `localStorage`에 저장하지 않음
- 학생 설문 임시 저장: 해당 탭의 `sessionStorage`, 최대 12시간

### 7.2 권한

- `anon`: 유효한 참여 토큰을 통한 학생 명단 조회와 새 응답 제출만 허용
- `authenticated`: 직접 테이블 접근이 아니라 담당 학급을 검사하는 보안 함수만 실행
- 다른 교사 계정: 담당하지 않은 학급의 명단·응답·관찰·분석·감사 로그 접근 불가

### 7.3 민감정보 처리

- 학생 이름과 설문 원문은 민감정보로 취급한다.
- PDF는 교사가 직접 생성하며, PDF와 백업 파일은 안전한 교사용 저장공간에만 보관한다.
- 로그와 파일럿 보고서에는 학생 이름, 응답 원문, 교사 이메일과 학급 ID를 포함하지 않는다.

## 8. 분석 안전 원칙

1. 단일 점수나 단일 응답으로 학생 상태를 확정하지 않는다.
2. 관계망의 선·집단·주변부는 실제 관계에 대한 가설로만 사용한다.
3. 학생이 직접 작성한 즉시 도움 요청은 계산 점수보다 먼저 확인한다.
4. 가능한 해석을 복수로 제시하고 교사가 확인할 장면과 질문을 함께 제공한다.
5. 결석, 미제출, 분석 제외와 응답자 구성 변화를 명시한다.
6. AI 결과와 규칙 기반 결과를 실제 관찰 기록과 구분해 저장한다.
7. 우선순위 숫자는 위험도, 생활평가 또는 학생 서열로 외부에 제공하지 않는다.

## 9. 비기능 요구사항

### 보안

- 모든 교사용 민감 RPC는 `auth.uid()`와 학급 소유권을 확인한다.
- 민감 테이블은 RLS를 활성화하고 `anon`, `authenticated`의 직접 접근을 철회한다.
- 비밀키, 개인 액세스 토큰과 실제 학생 자료를 저장소에 커밋하지 않는다.

### 접근성과 반응형 UI

- 핵심 터치 영역은 최소 44px을 확보한다.
- 360px 이상 모바일, 태블릿과 데스크톱에서 주요 기능을 사용할 수 있어야 한다.
- 색상만으로 상태를 구분하지 않고 텍스트 레이블을 함께 제공한다.

### 신뢰성

- 학생 제출은 고유 제출 ID를 사용해 재시도 중복을 방지한다.
- DB 변경은 순서가 명확한 SQL 마이그레이션으로 관리한다.
- 분석 핵심 계산과 개인정보 경계는 자동 테스트로 검증한다.
- GitHub Pages 배포 버전은 캐시 버전을 명시한다.

### 성능

- 일반적인 학급 규모 15~35명과 12개월 응답을 기준으로 핵심 화면을 브라우저에서 원활히 표시한다.
- AI 결과는 저장해 다시 사용하고 불필요한 API 재호출을 방지한다.

현장 파일럿 전에는 목표값이며 학생 평가 지표로 사용하지 않는다.

## 11. 수용 기준

v1.2 릴리스 후보는 다음 조건을 모두 만족해야 한다.

1. 로그인 전 학생 이름·응답·교사 이름·학급 ID가 표시되지 않는다.
2. 다른 교사 계정으로 운영 학급 RPC를 호출하면 거부된다.
3. 학생 참여 토큰 없이 명단 조회와 응답 제출이 불가능하다.
4. 동일 제출을 두 번 호출해도 같은 응답 ID와 행 1개만 유지된다.
5. 안전 신호 상태 변경 후 새로고침해도 상태·메모·확인일이 유지된다.
6. 안전 신호 변경 내용이 감사 로그에 기록된다.
7. 학생 상세에 해당 학생의 설문·신호·관찰만 표시된다.
8. 결석·미제출 월이 관계 평균에 0점으로 포함되지 않는다.
9. AI 결과가 한국어로 표시되고 검증 가능한 근거 원문과 화면 하단의 안전 안내가 연결된다. 일반적인 분석 한계 문장은 학생별 기본 카드에서 반복하지 않는다.
10. PDF가 학교 브라우저에서 파일로 다운로드되고 한글이 깨지지 않는다. *(2026년 8월 중순 이후 현장 검증)*
11. 3~5대 태블릿 동시 제출 후 교사 제출 현황에 모두 반영된다. *(2026년 8월 중순 이후 현장 검증)*
12. 백업 후 시험 학급 복구가 성공하고 운영 학급 데이터가 변경되지 않는다.
13. 자동 테스트가 모두 통과하고 공개 저장소에 비밀키·학생 자료가 없다.
