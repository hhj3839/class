# 우리반 이음 담임 운영 가이드

- 운영 주소: https://hhj3839.github.io/class/
- 학생 참여: `설문 관리` → `학생 설문 링크`
- 현장 점검 기준: [v1.2.1 현장 파일럿 점검표](FIELD_PILOT_v1.2.1.md)

## 수업 전 준비

1. 운영 주소에서 **교사 로그인** 후 담당 학급과 학생 명단이 맞는지 확인합니다.
2. `설문 관리`의 **학생 설문 링크** 탭에서 QR 또는 링크를 준비합니다.
3. 링크에 `?join=` 토큰이 있는지 확인합니다. 새 링크를 만들면 이전 링크는 즉시 사용할 수 없습니다.
4. 태블릿 1대로 학생 화면을 열어 명단과 첫 문항이 표시되는지 미리 확인합니다.
5. 실제 운영 전 JSON 백업을 만들고 학교의 안전한 교사용 저장공간에 보관합니다.

## 학생 참여 진행

1. 학생에게 [학생 설문 참여 안내](STUDENT_SURVEY_GUIDE.md)와 QR 또는 링크를 보여 줍니다.
2. 학생이 **자기 이름**을 선택했는지 확인하되 응답 화면은 들여다보지 않습니다.
3. 제출 완료 화면이 나올 때까지 브라우저 탭을 닫지 않도록 안내합니다.
4. `설문 관리` → **제출 현황**에서 설문 달을 확인하고 **제출 현황 새로고침**을 누릅니다.
5. 제출 인원과 미제출 학생을 확인하되 미제출을 0점이나 부정적 상태로 해석하지 않습니다.

## 제출 후 살펴보기

1. `우리 반 현황`에서 이번 달 참여와 변화를 확인합니다.
2. `학생별 살펴보기`에서 관계 점수·긍정 언급·지난 응답과 비교를 먼저 봅니다.
3. `학급 분석`의 AI 내용은 판단이나 진단이 아닌 **확인할 장면과 대화를 정하는 참고 자료**로만 사용합니다.
4. 확인이 필요한 내용은 학생 원문만으로 단정하지 말고 교실 장면, 비공개 대화와 보호요인을 함께 확인합니다.
5. PDF와 백업에는 학생 개인정보가 포함될 수 있으므로 교사용 안전한 저장공간에만 보관하고 메신저로 전달하지 않습니다.

## 문제가 생겼을 때

| 상황 | 바로 할 일 |
|---|---|
| 학생 명단이 보이지 않음 | 주소에 `?join=`이 있는지 확인하고 설문 관리의 최신 링크를 다시 엽니다. |
| 작성 중 연결이 끊김 | 탭을 닫지 않고 Wi-Fi 연결 후 같은 화면에서 다시 시도합니다. |
| 제출 완료 여부가 불분명함 | 학생 화면을 반복 제출시키기 전에 제출 현황을 새로고침해 확인합니다. |
| 다른 교사·로그아웃 화면에 학생 자료가 보임 | 즉시 사용을 중단하고 P0 오류로 기록합니다. |
| 즉시 도움 요청이 확인됨 | 앱 시험보다 학교의 기존 학생 보호·보고 절차를 우선합니다. |
| 오류 화면을 기록해야 함 | 학생 이름과 응답 원문을 가리고 [현장 오류 기록 양식](FIELD_ISSUE_TEMPLATE.md)을 사용합니다. |

## 종료 전 확인

- [ ] 제출 현황과 실제 참여 인원이 일치한다.
- [ ] 도움 요청과 확인이 필요한 응답을 놓치지 않았다.
- [ ] 로그아웃한 뒤 뒤로 가기·새로고침에서 학생 자료가 보이지 않는다.
- [ ] 다운로드한 PDF·백업을 안전한 위치로 옮기거나 불필요한 시험 파일을 삭제했다.
- [ ] 발견한 문제의 심각도, 재현 조건과 재시험 결과를 기록했다.
