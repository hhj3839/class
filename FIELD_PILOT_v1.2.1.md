# 우리반 이음 v1.2.1 현장 파일럿 점검표

- 기준 버전: `v1.2.1-prepilot`
- 기준 커밋: `aa5e944`
- 대상 시기: 2026년 8월 중순 이후
- 배포 주소: https://hhj3839.github.io/class/

## 1. 시험 원칙

- 실제 학생 응답은 담당 담임만 확인하고 화면 사진·오류 기록에는 이름과 원문을 남기지 않는다.
- 오류 재현에는 `태블릿 A`, `학생-테스트1`처럼 비식별 표기를 사용한다.
- 즉시 도움 요청이 포함된 실제 응답은 시험 자료로 복제하지 않고 학교의 기존 보호 절차에 따라 처리한다.
- 시험 전 백업을 만들고 시험용 제출과 운영 제출을 구분한다.

## 2. 준비물과 역할

- [ ] 학생용 태블릿 3~5대
- [ ] 담임교사용 PC 또는 태블릿 1대
- [ ] 인쇄 가능한 교사용 기기 1대
- [ ] 안정적인 Wi-Fi와 연결 중단 시험이 가능한 기기 1대
- [ ] 유효한 학생 참여 QR 또는 링크
- [ ] 담당 담임 계정과 접근 권한이 없는 다른 교사 시험 계정
- [ ] 시험 전 Supabase JSON 백업
- [ ] [오류 기록 양식](FIELD_ISSUE_TEMPLATE.md)

역할은 진행 교사 1명, 학생 역할 3~5명, 결과 기록자 1명으로 나누는 것을 권장한다.

## 3. 시험 전 점검

- [ ] 배포 사이트가 열리고 `v1.2.1-prepilot` 기준 기능이 표시된다.
- [ ] 담임 계정으로 로그인할 수 있다.
- [ ] 로그아웃 상태에서는 학생 이름·응답·교사 이름·학급 ID가 보이지 않는다.
- [ ] 학생 참여 링크가 `?join=` 임의 토큰을 사용한다.
- [ ] QR과 링크가 동일한 학생 설문으로 연결된다.
- [ ] 시험용 제출을 구분할 시간대와 학생 역할 번호를 기록했다.
- [ ] 시험 전 백업 파일을 교사용 안전한 저장공간에 보관했다.

## 8. 중단 기준

다음 중 하나가 발생하면 추가 시험을 중단하고 P0 오류로 기록한다.

- 다른 교사나 로그인 전 화면에 학생 개인정보가 노출됨
- 제출 완료 안내가 나왔지만 응답이 저장되지 않음
- 한 번의 제출로 중복 행이 생성됨
- 실제 운영 학급 자료가 시험 복구 과정에서 변경됨
- 즉시 도움 요청이 교사 화면에 반영되지 않음
